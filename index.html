Local-first task manager for humans and agents

人とAIが同じタスクを扱える、ローカル完結型タスク管理。

Cotaskaは、タスクをMarkdownファイルとして保存するWindows向けデスクトップアプリです。人が画面で整理し、AIエージェントがファイルとして読み書きできる、これからの作業管理を目指しています。

Concept

クラウドに預ける前に、まず自分の手元で整える。

Cotaskaが大事にしているのは、タスクを「アプリの中だけのデータ」にしないことです。タスクはローカルPC上のMarkdownファイルとして保存され、一覧表示用のインデックスはアプリが生成します。

だから、普段は普通のタスク管理アプリとして使いながら、必要なときには開発者やAIエージェントが同じファイルを直接読めます。人間の操作感と、AIが扱いやすいデータ構造を両立するための設計です。

01

Local first

タスクデータはローカルPC上に保存。ネットワーク接続や外部サービスを前提にせず、個人利用や小規模な開発作業で扱いやすい形にしています。

02

Markdown as source

タスクは個別のMarkdownファイルで管理。タイトル、状態、期限、リスト、本文メモを、読みやすく追跡しやすい形式で残せます。

03

Calm task flow

今日、明日、次の7日間、リスト、完了、ごみ箱。日々の作業でよく使う視点に絞り、整理より実行に戻りやすい画面を目指しています。

04

Built for agents

AIがタスク一覧を読み、対象タスクを開き、作業結果をMarkdownに反映しやすい構成です。人間とAIの共同作業を前提にしています。

Human and AI workflow

画面で見る。ファイルで渡す。AIに手伝ってもらう。

Cotaskaは、人が見やすいUIと、AIが読みやすいファイル構造の間に橋をかけます。タスク管理を「人間だけの整理」から「AIと共有できる作業状態」へ広げます。

Positioning

Cotaskaが目指す位置。

一般的なタスク管理

アプリのUIで使いやすい一方、AIや外部ツールからはデータの中身を直接扱いにくいことがあります。

ただのMarkdown管理

自由度は高い一方、一覧、期限、状態、並び替えなどの日常的な操作は自分で整える必要があります。

Cotaska

タスク管理アプリとして使いながら、タスクの正本はMarkdownファイルとして残す。人にもAIにも開いた形を目指します。
